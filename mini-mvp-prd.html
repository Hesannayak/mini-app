Feature 1 —
Voice UPI Payments

The core habit loop. Say it — Mini does it — confirms back. Works in Hindi, Tamil, Telugu from Day 1.

🎤 F1 — Voice UPI Payments P0 · Must Ship

User Story
As a user, I want to send money to someone by speaking naturally in Hindi/Tamil, so that I don't need to type UPI IDs or navigate menus.
Acceptance Criteria
1. User speaks command → Mini transcribes in <500ms
2. Intent extracted with >92% accuracy for top 20 commands
3. Confirmation shown before any payment
4. Payment executes via Razorpay SDK
5. Voice confirmation plays on success
6. Transaction logged to spending history
Languages (MVP)
Hindi, Tamil, Telugu. English fallback always available.
Supported Commands
"Maa ko 500 bhej do" · "Check balance" · "Seedha UPI se bhej" · "Kitna bacha hai?" · "Last 5 transactions dikhao" · "Request money from Rahul"
Payment Rail
Razorpay UPI SDK (P2P via NPCI — zero MDR). Razorpay PA license for merchant payments Phase 2.
Safety Rules
Always confirm before executing. New contacts → mandatory confirm. Amount >₹2,000 → biometric confirm. Night hours (12am–5am) → always confirm. Max ₹10,000/transaction MVP.
Offline Behavior
Voice captured and queued. "Internet aane pe bhejega Mini" — notifies user. Draft stored locally.
Error Handling
Insufficient balance → voice alert in Hindi. Bank downtime → "Bank server busy hai, 2 minute mein try karo". Failed payment → never double-charge. Full refund within 1 minute.
US-001
As Rahul, I say "Paisa bhej do Maa ko, 500 rupay" and Mini identifies Maa from my contacts, shows a confirm screen, and sends ₹500 with one tap.

P0

P1

Voice NLP Implementation Note

Use Sarvam AI's Saaras ASR model for Indian language speech-to-text — built specifically for Indian accents and background noise. Significantly outperforms Whisper on Hinglish and regional languages. Fine-tune on financial vocabulary (UPI, paisa, rupay, bhej, account). Entity extraction via custom NER model trained on 50,000 Indian financial utterances.

Feature 2 —
Daily Spending Intelligence

Every transaction categorized, every pattern surfaced. Delivered as a conversational daily summary — not a dashboard.

📊 F2 — Daily Spending Intelligence P0 · Must Ship

Data Sources
1. SMS bank alerts (parsed on-device, no server upload)
2. Account Aggregator via Setu / Finvu (explicit consent)
3. Mini's own UPI transactions (auto-logged)
Categories
Food & Dining · Transport · Groceries · Bills & Utilities · EMI/Loans · Entertainment · Shopping · Health · Education · Family Transfers · Other
Daily Summary
Pushed at 9pm every day. Voice + text. "Aaj aapne ₹1,240 kharch kiye — ₹480 khane pe, ₹340 auto mein, ₹420 groceries." Tappable to drill down.
Weekly Report
Sunday 10am. Compared to last week. Budget progress. Category breakdown. Biggest spend. Savings this week.
Smart Alerts
Budget 80% used → alert. Unusual spike detected → alert. Merchant overcharge pattern → alert. Duplicate transaction → immediate alert.
Privacy
SMS parsed entirely on-device. Raw SMS never leaves phone. Only structured {amount, merchant, category, timestamp} synced to server. Encrypted in transit.

Categorization AI — Implementation

Train a lightweight BERT-based classifier on 200,000 Indian bank SMS messages (synthetic + real, anonymized). Categories mapped to Indian merchant names — "Swiggy", "IRCTC", "BigBasket", "Zepto", kirana store names in Tamil script, etc. Model runs on-device (<5MB) for privacy. Falls back to keyword rules when confidence <70%. User can correct categories — corrections feed back into model via federated learning.

Feature 3 —
Proactive Bill Manager

Never miss a bill again. Mini tracks, reminds, and pays — all via voice.

📋 F3 — Proactive Bill Manager P1 · Must Ship

Bill Types
Electricity (all state boards via BBPS) · Mobile prepaid/postpaid · Internet/broadband · Gas (PNG/LPG) · Water bill · OTT subscriptions · Insurance premiums · Loan EMIs · School fees · Rent
Detection
Auto-detect from SMS patterns. "MSEDCL bill of ₹1,840 due on 15th" parsed and added to bill tracker automatically. User confirms on first detection.
Reminders
7 days before · 3 days before · 1 day before · Day of (morning 8am) · Overdue (same day 6pm). All in user's language. Push + WhatsApp fallback.
Payment
Via Setu BBPS integration. "Bijli ka bill bharo Mini" → shows amount → user says "haan" → paid. Setu handles all BBPS biller connections (800+ billers).
Smart Features
Bill amount anomaly detection (>20% spike → alert). Due date prediction for irregular bills. "Aapka bill last 3 mahine mein badh raha hai" pattern alerts.

Feature 4 —
AI Savings Coach

A financially intelligent friend who knows your transactions and helps you improve — conversationally, in your language.

🤖 F4 — AI Savings Coach P1 · Must Ship

LLM Backend
Claude claude-sonnet-4-20250514 via Anthropic API. System prompt: financial coach persona, always responds in user's language, never gives specific investment advice, stays behavioural.
Context Window
Last 90 days transactions + user profile + current Mini Score + active goals + bill history. Compressed to <4,000 tokens per query.
Proactive Insights
Pushed 3x/week. "Is haffte Swiggy pe 40% zyada kharch hua. Ghar pe banana try karo — ₹600 bachenge mahine mein." Always actionable. Always quantified.
Conversational
User can ask anything financial: "Mujhe ₹50,000 emergency fund banana hai" → Mini creates a savings plan based on their actual income/expense pattern.
Goal Tracking
User sets goals ("iPhone ke liye bachana hai — ₹80,000"). Mini tracks weekly progress, adjusts suggestions if off-track, celebrates milestones.
Hard Guardrails
Never: specific stock/fund recommendations · loan amount advice · insurance product advice · tax filing advice. These trigger "iske liye expert se baat karo" response.

The "Friend, Not Advisor" Principle

Every Coach response must feel like a smart friend's advice, not a financial advisor's report. "Yaar, tune is mahine bahut kharch kiya Zomato pe" is correct tone. "Your food expenditure of ₹4,234 exceeds recommended budgetary allocation by 34%" is wrong tone. All LLM outputs go through a tone validator before display. This is product-critical — wrong tone kills retention.

Feature 5 —
Mini Score™

One number that replaces financial anxiety. 0–100. Updated weekly. Explained in plain language.

⭐ F5 — Mini Score™ P1 · Must Ship

Score Range
0–100. Buckets: 0–40 (Starting Out 🌱) · 41–60 (Getting Better 📈) · 61–75 (Doing Well ✅) · 76–90 (Excellent 🌟) · 91–100 (Mini Champion 🏆)
Components
Bill Payment Discipline (30%) · Spending Control vs Budget (25%) · Savings Rate (25%) · Income Stability (20%)
Scoring Algorithm
Bill on time = +3pts/bill. Late = -5pts. Missed = -10pts. Under budget = +5pts/week. Savings rate >10% = +8pts. Income regular = +5pts. Score decays 1pt/week without activity.
Update Frequency
Weekly. Every Sunday 9am. With voice explanation: "Mini Score 74 ho gaya — 3 point badha! Bijli bill time pe bhari isliye."
What It Unlocks
Score >60: Mini Pro features unlocked. Score >70: Pre-approved insurance offers. Score >80: Partner loan pre-approval (Phase 2). Score badges shareable (WhatsApp).
Gamification
Weekly streak counter. "7-week bill payment streak 🔥". Monthly improvement badge. "Top 10% in your city" social proof (anonymous cohort comparison).

Database Schema

PostgreSQL with TimescaleDB for time-series transactions. Redis for sessions and real-time. All PII encrypted at column level.

Backend Architecture

Microservices-lite. Not over-engineered. 5 focused services that can be maintained by a 4-person backend team.

1

API Gateway (Kong / AWS API Gateway)

Rate limiting · JWT validation · Request routing · SSL termination · CORS. All mobile requests hit gateway first. 10,000 req/min per service limit MVP.

2

Voice Service (Node.js + Python)

Receives audio → calls Sarvam AI ASR → returns transcript → runs intent classification (custom model) → extracts entities → returns structured intent. Target: <800ms total. WebSocket for real-time streaming.

3

Payments Service (Node.js)

Wraps Razorpay SDK. Handles UPI initiation, confirmation, status polling. Idempotency keys on all payment requests (never double-charge). Webhook receiver for Razorpay callbacks. Setu BBPS for bill payments.

4

Intelligence Service (Python / FastAPI)

Transaction categorization ML · SMS parser · Mini Score calculator · Spending summary generator · Budget alert logic. Runs as async workers. Processes events from Kafka queue. No user-facing latency dependency.

5

Coach Service (Python)

Manages conversations with Claude API. Builds context window from user's transaction history. Enforces guardrails (no investment advice). Generates proactive insights 3x/week. Stores conversation history in Redis (30-day TTL).

6

Notification Service (Node.js)

Firebase FCM for push · WhatsApp Business API for WhatsApp · SMS via Twilio/Kaleyra fallback. Bill reminder scheduler (cron). Daily summary trigger 9pm per user timezone. Templated in 12 languages.

voice-intent-handler.ts · Voice Service
interface VoiceIntent { intent: string; // 'send_money' | 'check_balance' | 'pay_bill' ... confidence: number; // 0.0 - 1.0 entities: { amount?: number; contact?: string; // resolved from user_contacts upi_id?: string; bill_type?: string; period?: string; // 'today' | 'this_week' | 'this_month' urgency?: 'high' | 'normal'; }; language: string; // detected language code raw_text: string; // ASR transcript } async function processVoiceCommand( audioBuffer: Buffer, userId: string ): Promise<AgentResponse> { // Step 1: ASR — audio to text const transcript = await sarvamASR(audioBuffer, { language: 'hi-IN', // auto-detect if unknown model: 'saaras-v2', domain: 'financial' // fine-tuned vocabulary }); // Step 2: Intent Classification const intent = await classifyIntent(transcript.text); // Step 3: Entity Resolution const resolved = await resolveEntities(intent, userId); // "Maa" → look up user_contacts WHERE name_aliases @> ['Maa'] // Step 4: Permission Check const allowed = await checkPermission(userId, resolved.intent, { amount: resolved.entities.amount, isNewContact: resolved.isNewContact }); // Step 5: Route to appropriate handler switch (resolved.intent) { case 'send_money': return handleSendMoney(resolved, allowed.requireConfirm); case 'check_balance': return handleCheckBalance(userId); case 'pay_bill': return handlePayBill(resolved, userId); case 'spending_summary': return handleSpendingSummary(resolved.entities.period, userId); default: return handleCoachQuery(transcript.text, userId); } }

  • React Native project, design system tokens, navigation skeleton
  • Auth service: OTP via Kaleyra, JWT, refresh tokens
  • PostgreSQL + TimescaleDB on AWS RDS. Schema v1 migrated.
  • CI/CD: GitHub Actions → ECS deploy on merge to main
  • Razorpay sandbox integrated. Test payments flowing.
WEEK 3–4 · Voice MVP
🎤 Voice Pipeline Live
  • Sarvam AI ASR integrated. Audio upload → transcript in <600ms.
  • Intent classifier v1: top 20 intents, Hindi only
  • Entity extractor: amount, contact name, time period
  • Voice screen built in React Native. Press-to-talk working.
  • WebSocket streaming for real-time transcript display
WEEK 5–6 · Payments
💸 UPI Payments End-to-End
  • Razorpay UPI SDK in React Native. Send/receive working.
  • Contact resolution: "Maa" → UPI ID lookup from user_contacts
  • Confirm screen with biometric auth (FaceID/fingerprint)
  • Transaction logged on success. Auto-categorized by ML model v1.
  • Balance check via Razorpay API. <2s response target.
  • Payment failure handling + retry logic
WEEK 7–8 · SMS Parsing
📱 Transaction Intelligence
  • On-device SMS parser (React Native Background Fetch)
  • Indian bank SMS regex patterns for 30 major banks
  • Structured transaction sync to server (amount, merchant, time)
  • Categorization ML model v1 deployed. 85%+ accuracy target.
  • Home screen: transaction list live from real SMS data
WEEK 9–10 · Spending Intelligence
📊 Daily Summary Live
  • Spending summary API: aggregated by category and period
  • Spending screen: category breakdown, month view, week view
  • Daily 9pm push notification with spending summary
  • Budget setup flow. Budget vs actual comparison.

The #1 Non-Negotiable Before Launch

Zero tolerance for financial loss due to app error. Every payment must have: idempotency keys, confirmation step, biometric auth for amounts >₹1,000, full audit trail, and a 24/7 Hindi-speaking support line. One viral story of "Mini ne galat jagah paise bhej diye" will kill the product. Get this right before acquiring a single user.
