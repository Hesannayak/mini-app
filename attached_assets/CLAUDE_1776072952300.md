## 🧭 What Is Mini?

**Mini** is India's first voice-first AI financial companion. A free mobile app (iOS + Android) where users speak naturally in Hindi, Tamil, or Telugu to manage their money — send UPI payments, track spending, pay bills, get savings coaching, and see their financial health score.

**The one-line pitch:** "Apni bhaasha mein, apna paisa samjho."

**What Mini is NOT:**
- Not a payments app competing with GPay/PhonePe on transactions
- Not a budgeting dashboard (no Excel-style tables)
- Not a robo-advisor or investment platform
- Not a bank

**What Mini IS:**
- A voice-first financial intelligence layer on top of UPI
- A daily habit app (target: 3 opens/day per user)
- A trust-building product that earns the right to recommend financial products later
- Built for Bharat — tier-2 cities, Hindi speakers, budget Android phones

## 🎯 MVP Features (Scope — Do Not Expand)

The MVP has exactly **5 features**. Do not add features not listed here without explicit approval.

### F1 — Voice UPI Payments `[P0 · Must Ship]`
Users send/receive money, check balance, and request payments via voice in Hindi/Tamil/Telugu.
- Payment rail: **Razorpay UPI SDK** (P2P = zero MDR by NPCI mandate)
- Always show confirm screen before executing any payment
- Biometric auth for amounts > ₹1,000
- Max transaction: ₹10,000 (MVP limit)
- Supported intents: send_money, check_balance, request_money, transaction_history

### F2 — Daily Spending Intelligence `[P0 · Must Ship]`
Auto-categorize all UPI transactions from SMS + Account Aggregator. Daily 9pm voice summary.
- SMS parsed **on-device only** — raw SMS never leaves the phone
- Only structured `{amount, merchant, category, timestamp}` syncs to server
- Categories: Food, Transport, Groceries, Bills, EMI, Entertainment, Shopping, Health, Education, Family, Other
- Daily push notification at 9pm with spending summary in user's language

### F3 — Proactive Bill Manager `[P1 · Must Ship]`
Track recurring bills, send reminders, pay via BBPS with voice.
- Bill payment rail: **Setu BBPS** (800+ billers)
- Reminders: 7 days, 3 days, 1 day before due
- Auto-detect bills from SMS patterns
- Voice command to pay: "Bijli bill bharo" → confirm → paid

### F4 — AI Savings Coach `[P1 · Must Ship]`
Conversational AI coach using Claude API. Behavioral nudges only — no investment advice.
- LLM: **Claude claude-sonnet-4-20250514** via Anthropic API
- Context: last 90 days transactions + Mini Score + goals (compressed to <4,000 tokens)
- Always responds in user's detected language
- **Hard guardrail:** Never recommend specific stocks, mutual funds, or loans. Trigger "iske liye expert se baat karo" for regulated advice.
- Proactive insights pushed 3x/week

### F5 — Mini Score™ `[P1 · Must Ship]`
Financial health score 0–100, updated weekly every Sunday.
- Components: Bill Discipline (30%) + Spending Control (25%) + Savings Rate (25%) + Income Stability (20%)
- Score buckets: 0–40 🌱 · 41–60 📈 · 61–75 ✅ · 76–90 🌟 · 91–100 🏆
- Weekly explanation in user's language: "Score 3 point badha kyunki bijli bill time pe bhari"
- No CIBIL/credit bureau integration in MVP — purely behavioral

## 🗄️ Database Schema

Full schema is in `schema.sql`. Key tables:

```
users               — user profiles, language pref, permission level, Mini Score
transactions        — TimescaleDB hypertable, all financial transactions
bills               — tracked recurring bills with BBPS biller IDs
mini_score_history  — weekly score snapshots with component breakdown
budgets             — user-defined category budgets
savings_goals       — user goals with target amounts and dates
user_contacts       — saved UPI contacts with name aliases ("Maa", "Amma")
voice_sessions      — audit trail of voice commands (transcript deleted after 30d)
```

**Critical schema rules:**
- All PII (phone, name, UPI IDs) encrypted at column level using AWS KMS
- `phone` stored encrypted, `phone_hash` (SHA-256) used for lookups
- Raw SMS content in `transactions.raw_sms` — deleted after 30 days via cron
- Voice transcripts in `voice_sessions.transcript` — deleted after 30 days
- Never store UPI PINs, bank passwords, or payment credentials. Ever.

## 🎤 Voice Pipeline

Every voice command flows through 7 steps:

```
1. Audio captured on device (React Native microphone)
2. Audio → Sarvam AI ASR → transcript (<600ms target)
3. Transcript → Intent Classifier → intent + confidence
4. Intent + context → Entity Extractor → structured entities
5. Entities → Permission Checker → auto-execute OR require confirm
6. Route to sub-handler (payments / bills / spending / coach)
7. Response → TTS voice output + screen update (<3s total target)
```

**Intent classification targets (MVP):**
```
send_money          — "Maa ko 500 bhej do"
check_balance       — "Kitna bacha hai?"
request_money       — "Rahul se 200 maango"
pay_bill            — "Bijli ka bill bharo"
spending_summary    — "Aaj kitna gaya?" / "Is hafte ka hisaab"
transaction_history — "Last 5 transactions dikhao"
set_budget          — "Khane pe 3000 ka budget set karo"
coach_query         — anything else → route to Claude
```

**Confidence thresholds:**
- > 0.90 → execute (known pattern) or show confirm
- 0.70–0.90 → always show confirm screen
- < 0.70 → "Mujhe samajh nahi aaya, dobara boliye" (ask to repeat)
- Never execute payment if confidence < 0.80

## 🔐 Permission Model

Users progress through 4 levels. Default is Level 1 for new users.

| Level | Name | Behavior | Unlocked |
|-------|------|---------|---------|
| 1 | Observe | Read-only. No payments. | Default |
| 2 | Confirm First | All actions need explicit confirm | After 7 days |
| 3 | Smart Auto | Known patterns auto-execute under threshold | After 30 days |
| 4 | Full Agent | Autonomous execution for approved categories | User opt-in only |

**Always confirm, regardless of level:**
- Any new UPI contact (never sent before)
- Amount > ₹2,000
- Between 12am–5am
- First time paying a bill

## 🛡️ Security Rules (Non-Negotiable)

1. **Never store UPI PINs** — Razorpay SDK handles pin entry in secure enclave
2. **SSL pinning** on all API calls from mobile app
3. **Biometric auth** required for amounts > ₹1,000
4. **Idempotency keys** on every payment request — never double-charge
5. **SMS parsed on-device** — raw content never uploaded to server
6. **All PII encrypted** at column level (AWS KMS)
7. **Audio deleted** from S3 within 60 seconds of transcription
8. **Transcripts deleted** from DB after 30 days (cron job)
9. **DPDP right to erasure** — `DELETE /api/v1/user/data` must wipe everything in 72h
10. **Data stays in India** — AWS Mumbai only. No cross-region data transfer for user PII.

## 🚨 Critical Rules — Read Before Every Task

1. **Never execute a payment without user confirmation.** This is the #1 product safety rule. Violations will be reverted immediately.

2. **Never store UPI PINs or bank credentials.** Razorpay SDK handles all sensitive payment auth.

3. **SMS parsing is on-device only.** The raw SMS string never leaves the user's phone. Only the structured extracted data syncs.

4. **All data stays in AWS Mumbai.** No data to any region outside India. RBI compliance.

5. **The Coach never gives investment advice.** Any response resembling specific financial product recommendations must trigger the "iske liye expert se baat karo" guardrail.

6. **APK size is a hard constraint.** If a library pushes APK over 15MB, find a lighter alternative or don't add it.

7. **Test on budget hardware.** Every feature must be tested on a Redmi 9A or equivalent (2GB RAM, Android 8). If it's slow there, it doesn't ship.

8. **Error messages are in the user's language.** No English error messages shown to Hindi/Tamil/Telugu users. Ever.

9. **Idempotency on all payments.** Every payment initiation must include an idempotency key. Double-charging a user is unacceptable.

10. **When in doubt, ask.** This is a fintech product. Wrong decisions have real financial consequences for real users. Ask before assuming.
